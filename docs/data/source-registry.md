# Free Public Data Source Registry

This registry tracks free public sources that can feed ORSGraph, NeighborOS, and CaseBuilder. It is intentionally docs-first: it records source intent, provenance requirements, connector priority, and graph mapping before any crawler or parser code is added.

The ingestion rule is fixed:

```text
API first.
Bulk download second.
Static crawl third.
Search-page scraping last.
```

Every connector must preserve raw artifacts, source metadata, content hashes, retrieval timestamps, license or terms notes, parser diagnostics, JSONL nodes, JSONL edges, and a QC report. Official sources should be separated from nonprofit and secondary sources in graph metadata.

Primary source references:

- Oregon Legislature OData: <https://www.oregonlegislature.gov/citizen_engagement/Pages/data.aspx>
- Oregon Legislature OData acceptable use agreement: <https://www.oregonlegislature.gov/citizen_engagement/Documents/OLODataAcceptableUseAgreement.pdf>
- Oregon Open Data developer resources: <https://data.oregon.gov/developers>
- Oregon Judicial Department case records guidance: <https://www.courts.oregon.gov/how/pages/find.aspx>
- CourtListener REST API: <https://www.courtlistener.com/help/api/rest/>
- GovInfo API: <https://www.govinfo.gov/features/api>
- Federal Register API: <https://www.federalregister.gov/developers/documentation/api/v1>

## Registry Fields

Each source row uses the field definitions in [source-registry.schema.md](source-registry.schema.md).

| Field | Meaning |
| --- | --- |
| `source_id` | Stable registry identifier used by connector configs and QC reports. |
| `source_type` | Primary access shape: `api`, `bulk`, `static_html`, `pdf`, `socrata`, `arcgis`, or `search_page`. |
| `access` | Access model: `free`, `free_key_required`, `public_search`, or `mixed`. |
| `official_status` | `official`, `nonprofit`, `secondary`, or `unknown`. |
| `preferred_ingestion_method` | First method to use under the API-first policy. |
| `fallback_ingestion_method` | Allowed fallback when the preferred method does not cover the needed data. |
| `connector_status` | `not_started`, `planned`, `partial`, `implemented`, `blocked`, or `deferred`. |
| `priority` | `P0`, `P1`, or `P2`. |

## Connector Design

This is the target abstraction. Do not add Rust types for this docs-only change until parser-specific code is ready to converge.

```rust
pub trait DataConnector {
    fn source_id(&self) -> &'static str;
    fn source_kind(&self) -> SourceKind;
    async fn discover(&self) -> Result<Vec<SourceItem>>;
    async fn fetch(&self, item: &SourceItem) -> Result<RawArtifact>;
    async fn parse(&self, artifact: &RawArtifact) -> Result<GraphBatch>;
    async fn qc(&self, batch: &GraphBatch) -> Result<QcReport>;
}
```

Every connector should emit:

```text
raw artifact
source metadata
content hash
retrieval timestamp
license/terms notes
parser diagnostics
JSONL nodes
JSONL edges
QC report
```

## QC And Provenance Rules

- Store `source_id`, source URL, retrieval timestamp, raw hash, normalized hash, parser profile, connector version, and official-status metadata for every artifact.
- Preserve official disclaimers as first-class source metadata, especially the Oregon Legislature ORS online-database disclaimer.
- Separate official, nonprofit, secondary, and unknown sources in graph metadata and UI citations.
- Only refetch changed artifacts when stable checksums, ETags, Last-Modified headers, or dataset update timestamps are available.
- Treat public search pages as targeted lookup sources, not broad ingestion sources, unless terms and technical access clearly allow bulk collection.
- Keep expired, future, and superseded rules available for history while excluding them from active filing-date checks.
- Fail QC if required source fields are missing, graph IDs are unstable, hashes are absent, or parser diagnostics exceed source-specific thresholds.
