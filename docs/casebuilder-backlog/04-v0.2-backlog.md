# 04 - V0.2 Backlog

V0.2 focuses on richer legal work product: motions, declarations, exhibits, export packets, and stronger authority retrieval.

## CB-V02-001 - Motion Builder
- Priority: P1
- Area: Motions
- Problem: Motion workflows are not implemented.
- Expected behavior: Build motion outline, legal standard, fact-to-standard map, argument, declaration support, exhibits, proposed order, citation check, and QC.
- Implementation notes: Support generic motion first, then specialized flows.
- Acceptance checks: User can create a generic motion draft from selected claims/facts/evidence/authority.
- Dependencies: V0 drafting, authority search, evidence matrix, fact checks.
- Status: Todo

## CB-V02-002 - Motion templates
- Priority: P2
- Area: Motions
- Problem: Different motion types need different required sections and standards.
- Expected behavior: Templates for motion to dismiss, summary judgment, compel, TRO/preliminary injunction, amend, default, sanctions, and generic motion.
- Implementation notes: Keep templates jurisdiction-neutral unless source rules are available.
- Acceptance checks: Template creates expected draft outline and required support checklist.
- Dependencies: `CB-V02-001`.
- Status: Todo

## CB-V02-003 - Declaration and affidavit builder
- Priority: P1
- Area: Declarations
- Problem: Users need witness statements tied to facts/evidence.
- Expected behavior: Build declarant identity, numbered factual statements, exhibit references, verification/signature block, and support checks.
- Implementation notes: Reuse Draft and Evidence nodes with declaration-specific draft kind.
- Acceptance checks: Declaration draft can be generated from selected facts and checked for support.
- Dependencies: V0 facts, evidence, drafts.
- Status: Todo

## CB-V02-004 - Exhibit builder
- Priority: P1
- Area: Exhibits
- Problem: Exhibit labels exist in data but no full exhibit workflow exists.
- Expected behavior: Assign exhibit numbers, generate exhibit list, link exhibits to facts/draft paragraphs, and build export packet order.
- Implementation notes: Add Exhibit node if CaseDocument `is_exhibit` becomes insufficient; preserve stable exhibit labels when packet order changes.
- Acceptance checks: Exhibit list matches document labels and draft references.
- Dependencies: V0 documents, evidence, drafts.
- Status: Todo

## CB-V02-005 - Filing packet builder
- Priority: P1
- Area: Export
- Problem: Export routes are deferred stubs.
- Expected behavior: Build packet containing primary draft, exhibits, exhibit list, declarations, proposed order, and QC summary.
- Implementation notes: Do not implement court e-filing yet; expose packet preview, status, warnings, and downloadable generated artifacts.
- Acceptance checks: Packet preview shows included files, order, warnings, and missing items.
- Dependencies: `CB-V02-003`, `CB-V02-004`.
- Status: Todo

## CB-V02-006 - DOCX export
- Priority: P1
- Area: Export
- Problem: Legal work product needs editable export.
- Expected behavior: Export drafts and exhibit lists to DOCX with basic pleading-friendly structure.
- Implementation notes: Use a document generation pipeline with visual/render verification.
- Acceptance checks: Generated DOCX opens and preserves headings, paragraphs, numbering, citations, and exhibit references.
- Dependencies: Stable draft model.
- Status: Todo

## CB-V02-007 - PDF export
- Priority: P2
- Area: Export
- Problem: Users need reviewable PDF output.
- Expected behavior: Export draft/packet PDFs with clear "not filed" status unless final.
- Implementation notes: Build after DOCX or from same rendering source.
- Acceptance checks: PDF renders without layout overlap and includes generated timestamp/status.
- Dependencies: `CB-V02-006`.
- Status: Todo

## CB-V02-008 - Reranked authority retrieval
- Priority: P1
- Area: Legal authority
- Problem: Authority search should rank source-backed ORS provisions by usefulness to matter facts/claims.
- Expected behavior: Rerank relevant provisions, definitions, deadlines, remedies, penalties, exceptions, and source notes, with currentness and scope warnings.
- Implementation notes: Use existing rerank service when configured and expose retrieval metadata, currentness state, source note IDs, and definition-scope caveats.
- Acceptance checks: Authority results show why they were recommended, what claim/element they support, whether they are current, and any scope/currentness warnings.
- Dependencies: V0 authority search and ORSGraph rerank config.
- Status: Todo

## CB-V02-009 - Element library by claim type
- Priority: P2
- Area: Claims/elements
- Problem: Users currently need to define elements manually or rely on demo data.
- Expected behavior: Claim templates include suggested elements and authority candidates.
- Implementation notes: Start with Oregon landlord/tenant templates if product priority remains ORS-focused.
- Acceptance checks: Selecting a claim type creates editable elements with authority links.
- Dependencies: Authority retrieval and curated templates.
- Status: Todo

## CB-V02-010 - Strategy board
- Priority: P2
- Area: Strategy
- Problem: Strategy signals are not gathered into a work board.
- Expected behavior: Claim strength, defense strength, evidence completeness, authority strength, deadline risk, damages support, settlement leverage notes, weakness map, and next tasks.
- Implementation notes: Use transparent scoring with source-backed explanations.
- Acceptance checks: Every score is explainable by facts/evidence/authority/task state.
- Dependencies: QC findings, evidence gaps, authority gaps.
- Status: Todo

## CB-V02-011 - Export package model and status API
- Priority: P1
- Area: Export
- Problem: Export needs durable package state rather than one-shot stub endpoints.
- Expected behavior: Add `ExportPackage` records with included draft IDs, exhibit IDs, generated artifact IDs, warnings, QC status, created timestamp, and download status.
- Implementation notes: Add routes for create package, get package status, regenerate package, and download generated DOCX/PDF artifacts.
- Acceptance checks: User can create an export package preview, refresh status after generation, download available artifacts, and see blocking QC warnings.
- Dependencies: `CB-V02-005`, `CB-V02-006`, `CB-V02-007`.
- Status: Todo

## CB-V02-012 - Exhibit numbering and packet ordering rules
- Priority: P1
- Area: Exhibits/export
- Problem: Exhibit references can become unreliable if labels shift during drafting or export.
- Expected behavior: Maintain stable exhibit labels, packet order, source document mapping, draft references, and regeneration warnings when labels conflict.
- Implementation notes: Use explicit exhibit records and avoid deriving labels only from document order.
- Acceptance checks: Reordering a packet does not silently change existing draft exhibit references; conflicts surface as QC/export warnings.
- Dependencies: `CB-V02-004`, `CB-V02-005`.
- Status: Todo

## CB-V02-013 - Export visual verification workflow
- Priority: P2
- Area: Export quality
- Problem: DOCX/PDF output needs layout verification before users rely on filing packets.
- Expected behavior: Generated DOCX/PDF artifacts are rendered or inspected for headings, numbering, citations, exhibit references, page breaks, and overlap.
- Implementation notes: Keep generated status as review-needed until basic render checks pass.
- Acceptance checks: Export tests fail on unreadable output, missing sections, broken numbering, or missing exhibit references.
- Dependencies: `CB-V02-006`, `CB-V02-007`.
- Status: Todo

## CB-V02-014 - Authority recommendation attaches by context
- Priority: P1
- Area: Legal authority
- Problem: Current recommendation behavior is search-like and does not use selected claim/element/draft context deeply enough.
- Expected behavior: Recommendation accepts context IDs and returns ranked authorities with support target, reason, currentness, scope, and attach action metadata.
- Implementation notes: Use `claim_id` and later `element_id`, `draft_id`, `paragraph_id`, or `sentence_id`; never attach automatically without user review.
- Acceptance checks: Recommending authority for a claim or paragraph explains the target and allows one-click attach through the V0 authority attach endpoint.
- Dependencies: `CB-V0-025`, `CB-V02-008`.
- Status: Todo
